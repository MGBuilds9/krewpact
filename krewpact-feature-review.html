Executive Summary

What is KrewPact?

KrewPact is MDM Group's unified operations platform — replacing spreadsheets, disconnected tools, and manual processes across all five divisions. It combines CRM, project management, estimating, field operations, and financial tracking in one system, connected directly to ERPNext for accounting.

Why We Are Building It

  • Fragmented tools: Sales in email, projects in spreadsheets, accounting in Sage, documents in OneDrive — no connection between them.
  • No visibility: Leadership cannot see pipeline, project status, or financials in one view without manual consolidation.
  • Manual handoffs: Data re-entered multiple times across systems — slow, error-prone, and costly.
  • Division isolation: Each division operates independently with no shared systems or cross-division reporting.

KrewPact uses a 13-role access model. The system automatically shows or hides features based on each person's assigned role. Before we configure the system, we need you to assign roles to every member of your team.

  • KrewPact Account → ERPNext Customer

  • KrewPact Estimate → ERPNext Quotation

Appendix D: Glossary

TermDefinition
KrewPactMDM Group's unified construction operations platform. CRM, estimating, projects, portals, RBAC. Built on Next.js + Supabase + Clerk + ERPNext.
ERPNextOpen-source ERP system replacing Sage. Finance, procurement, and inventory source of truth. Hosted on MDM infrastructure, accessible via erp.mdmgroupinc.ca.
SupabaseManaged PostgreSQL cloud database. Stores all KrewPact operational data (workflows, field ops, audit trails, portals).
ClerkAuthentication provider. Handles login, M365 SSO, and user session management.
RBACRole-Based Access Control. The system of 13 roles that determines what each user can see and do in KrewPact.
BoldSignWhite-label electronic signature API. Used for contract signing in Phase 1.
BullMQBackground job queue system. Handles ERPNext sync, email sending, and other async tasks reliably with retry logic.
Cloudflare TunnelSecure connection between Vercel (cloud) and ERPNext (MDM servers). No VPN or port-forwarding required.
P0 / P1 / P2Phase 1 (Core), Phase 2 (Growth), Phase 3 (Scale). Defines the rollout order of features.
PIPEDACanada's federal privacy law. Governs how personal data is collected, used, and stored. KrewPact complies via Clerk (auth) and Supabase (data).
Construction ActOntario's Construction Act (2019 amendments). Governs lien rights, holdbacks, and prompt payment. KrewPact P1 includes holdback and lien period tracking.
WSIBWorkplace Safety and Insurance Board. Ontario workers' compensation. Trade partners must provide WSIB clearance certificates before site assignment.
CORCertificate of Recognition. Safety program certification for contractors. Tracked in Trade Compliance module (Scale).
CCDCCanadian Construction Documents Committee. Standard contract forms used in Canadian construction. KrewPact contract templates will align with CCDC formats.
RFIRequest for Information. Formal question from contractor to architect/engineer during construction. Tracked in Epic 10.
SubmittalProduct data sheets or shop drawings submitted to the architect/engineer for review and approval before installation.
Substantial CompletionThe point at which a project is sufficiently complete for its intended use. Triggers lien period and holdback release under Ontario Construction Act.
HoldbackAmount withheld from contractor payments (typically 10% in Ontario) until lien period expires after substantial completion.
ADPAutomatic Data Processing — MDM's payroll processor. KrewPact P1 will export approved timesheets to ADP.
SageMDM's current accounting software (Sage 50 + Sage Construction Mgmt). Being replaced by ERPNext. KrewPact Epic 21 covers the migration.
ResendTransactional email API. KrewPact uses Resend to send automated notifications and user emails.
PWAProgressive Web App. Web application that can be installed on mobile devices and works offline. KrewPact P2 field ops module will be a PWA.
