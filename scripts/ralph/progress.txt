- **Learnings for future iterations:**
  - Zod 4.3.6: `z.record(z.unknown())` crashes at runtime with "Cannot read properties of undefined (reading '_zod')". Use `z.record(z.string(), z.any())` instead for flexible JSON objects (like billing_address, shipping_address).
  - Zod 4.3.6 UUID validation is strict RFC 4122: rejects `00000000-0000-0000-0000-000000000001` (invalid version byte at position 13). Test fixture UUIDs like TEST_DIVISION_ID won't pass z.string().uuid(). For validator tests, use proper v4 UUIDs like `a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11`. The nil UUID `00000000-...00000` is accepted.
  - Zod 4 type inference: use `z.infer<typeof schema>` (TypeScript-only, not a runtime value). Works identically to Zod 3 syntax.
  - Lead stages (DB enum): new, qualified, estimating, proposal_sent, won, lost
  - Opportunity stages (DB enum): intake, site_visit, estimating, proposal, negotiation, contracted, closed_lost
