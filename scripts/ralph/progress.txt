## Codebase Patterns
- **Stack:** Next.js 16, React 19, TypeScript, Supabase, Clerk auth, Tailwind + shadcn/ui
- **Package manager:** npm (not pnpm/bun)
- **Test framework:** Vitest with jsdom, setup in __tests__/setup.ts
- **Test commands:** npm run test (Vitest), npm run lint (ESLint), npm run typecheck (tsc --noEmit), npm run build (next build)
- **Baseline tests:** 3,092 tests across 274 files (as of Phase 1 completion)
- **Test pattern for API routes:** Mock Clerk auth via vi.mock('@clerk/nextjs/server'), mock Supabase via vi.mock('@/lib/supabase/server'), test route handler functions directly
- **Shared test helpers:** Import from `@/__tests__/helpers` — mockSupabaseClient, mockClerkAuth, mockClerkUnauth, makeRequest, makeJsonRequest, factory functions
- **Mock order matters:** vi.mock() calls BEFORE imports
- **Context pattern:** `{ params: Promise.resolve({ id }) }` for Next.js 15 dynamic routes
- **Rate limit mock:** `vi.mock('@/lib/api/rate-limit', () => ({ rateLimit: vi.fn().mockResolvedValue({ success: true }), rateLimitResponse: vi.fn() }))`
- **Portal routes:** Need multi-table mocking (portal_accounts → portal_permissions → data table)
- **Zod UUIDs:** z.string().uuid() rejects non-v4 UUIDs — always use valid v4 in tests
- **Console.log migration:** All API/lib files migrated to structured logger. Error boundaries still use console.error (US-017 will fix)
- **Error boundaries:** 11 error.tsx files exist across all route groups
- **CommandPalette:** Currently only searches CRM entities — needs expansion (US-008)
- **useKeyboardShortcuts hook:** Exists, can be extended for chord shortcuts
- **useNotifications hook:** CRUD hooks exist but no Realtime subscription
- **Supabase client:** lib/supabase/server.ts creates server client, lib/supabase/client.ts for browser
- **Rate limiting:** All 259 non-cron/webhook routes have Upstash rate limiting
- **Sentry:** sentry.client.config.ts, sentry.server.config.ts, sentry.edge.config.ts all exist
- **Logger:** lib/logger.ts with structured logging (info, warn, error, debug)
- **Supabase Realtime:** `hooks/useRealtimeSubscription.ts` — generic hook for postgres_changes subscription. Accepts table, filter, onEvent, queryKeys, enabled.
- **React Compiler lint:** Self-referencing in `useCallback` is banned. For recursive patterns (reconnect), define function inside `useEffect` and store ref.
- **jsdom FormData bug:** `Request.formData()` hangs on streamed FormData bodies in jsdom. Fix: `vi.spyOn(req, 'formData').mockResolvedValue(formData)`.
- **Supabase JS v2.98.0** — full Realtime support via `.channel().on('postgres_changes', ...).subscribe()`
- **PATCH /api/notifications/[id] schema bug:** The route validated `{ read: boolean }` but the hook sends `{ state: 'read' }`. Always check that Zod schema matches the client-side mutation payload.
- **Popover component:** Use `@/components/ui/popover` (Popover, PopoverContent, PopoverTrigger) for dropdown panels.
- **date-fns available:** `formatDistanceToNow` from `date-fns` for relative timestamps.
- **@react-pdf/renderer:** Use `renderToBuffer()` server-side. Templates use `React.createElement` (no JSX in .ts files). Pass `new Uint8Array(buffer)` to `NextResponse` (Buffer type incompatible with BodyInit).
- **PDF generator pattern:** `lib/pdf/generator.ts` → `generatePdf(template, data)`. Templates in `lib/pdf/templates/`. Types in `lib/pdf/types.ts`.
- **Toast system:** `hooks/use-toast.ts` exports `toast()` and `useToast()`. Use `variant: 'destructive'` for errors. Also `components/ui/toast.tsx` exists.
- **Blob download pattern:** Use `fetch → blob() → URL.createObjectURL → anchor.click → revokeObjectURL`. Don't spy on `document.createElement` globally in tests — intercept only the 'a' tag to avoid breaking React rendering.
