## 2024-05-24 - QStash Webhook Authentication Bypass
**Vulnerability:** API routes (`app/api/executive/knowledge/embed/route.ts` and `app/api/executive/metrics/refresh/route.ts`) intended to be triggered by QStash were only checking for the *presence* of the `upstash-signature` header (`if (qstashSignature)`), allowing complete authentication bypass by sending an arbitrary signature value.
**Learning:** Checking for the presence of a signature header is insufficient; it must be cryptographically verified using the shared secret and the raw request body. Also, reading the body to verify the signature (`req.text()`) consumes the stream, requiring subsequent parsing from the returned string (`JSON.parse(rawBody)`).
**Prevention:** Always use `verifyQStashSignature` from `@/lib/queue/verify` (or equivalent cryptographic validation) to verify incoming webhook requests before granting access. Ensure the raw body is used for verification and then parsed for downstream logic to prevent stream consumption errors.
