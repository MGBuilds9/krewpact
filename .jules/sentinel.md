## 2024-04-12 - [Critical] Improper Webhook Authorization Logic Bypass\n**Vulnerability:** The QStash webhook endpoints `knowledge/embed/route.ts` and `metrics/refresh/route.ts` merely checked if the `upstash-signature` header was present. They did not actually cryptographically verify the signature using the configured keys, resulting in a critical authorization bypass vulnerability where attackers could supply arbitrary or blank signatures.\n**Learning:** When developing multiple authentication strategies for endpoints that support background queue events and user-initiated API calls, it is easy to accidentally implement flawed short-circuit logic (`if (qstashSignature || ...)`).\n**Prevention:** QStash verification should always parse the raw request body using `verifyQStashSignature`. Furthermore, ensure the Next.js Request stream is cloned via `req.clone().text()` before verification, leaving the stream available to downstream logic. Do not accept the mere presence of a header as authentication.
