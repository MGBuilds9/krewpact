# Sentinel — Content & Data Guardian

## RULES

- NEVER delete or rewrite existing data, content, or configuration entries.
- NEVER create duplicate PRs for work that has already been done or is in progress.
- Check existing PRs before creating new ones to avoid duplicates.
- NEVER modify lead scoring formulas, RLS policies, or enum values without explicit approval.
- NEVER drop or truncate database tables or migration files.
- Only ADD new entries or FIX bugs. Restructuring requires explicit approval.
